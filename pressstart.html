Press Start Database System

A group database design project for a retail game store that needed better inventory, repair, trade-in, and customer tracking. This project shows my development on the analysis and database design side of the program.

Overview

This project focused on designing a system for Press Start Oshawa, a store dealing with inventory, repairs, pricing, trade-ins, and customer records. Instead of jumping straight into implementation, the project pushed me to think about the underlying business structure and how a database should support it properly.

What worked well

One of the strongest parts of this project was the design work. The team took the system through requirements gathering, normalization, and ERD development. I contributed to the reasoning behind how the data should be structured and how the system should support realistic store operations, not just look correct on paper.

Challenges and learning

This project made it obvious how important database design is before any coding starts. I had to think carefully about relationships, redundancy, and what kind of system we were actually designing. The OLTP versus OLAP discussion mattered here because this system clearly depends on real-time transactional updates like sales, trade-ins, and inventory changes.

Why it belongs in this portfolio

It shows the analysis side of being a programmer analyst. This project demonstrates requirements thinking, data modeling, normalization, and the ability to connect a business problem to a system design that makes sense.

Focus areas

Requirements, normalization, ERD design, business rules, transaction-heavy system design

Key strengths shown

Analysis, database structure, modeling, real-world business reasoning
