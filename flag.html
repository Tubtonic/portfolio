Flag Guessing Game

An earlier C# game project that still matters in this portfolio because it highlights both technical progress and how much my development habits improved over time.

Overview

This was a C# game where a player is shown a flag and has to guess the correct country from multiple choices. The project itself is simpler than the others in this portfolio, but it still deserves a place because it represents an earlier stage in my development and the lessons from it stayed with me.

What worked well

The game had solid functionality for the scope. It displayed random flag images, kept score, included timers, hints, result screens, and some additional features like voice recognition. The core mechanics worked well and showed that I could already break a project into several working pieces.

Challenges and learning

The biggest issue was not the code itself. It was how the group handled version control and collaboration. We did not set the project up properly, and that caused problems later. Looking back, this taught me a lot about the importance of shared repositories, better planning, and cleaner team workflow.

Why it belongs in this portfolio

This project is useful because it shows growth. It gives context for why later projects were handled more professionally and why I became much more serious about version control, planning, and project structure.

Technologies

C#, game logic, UI, multimedia features

Key strengths shown

Early problem solving, feature integration, reflection, growth through mistakes
