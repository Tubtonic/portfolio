KIT – Keep In Touch Relationship Manager

A capstone project built to track interactions, manage reminders, and show who a user should reach out to next. This project matters most in my portfolio because it brought frontend, backend, authentication, database design, and real data flow together in one system.

Overview

KIT is a mobile relationship manager built with Flutter on the frontend and a Node.js and Express API on the backend, with Prisma handling the database layer. The core idea was to make something that feels genuinely useful while also forcing me to build a complete system instead of just one isolated feature.

What worked well

The strongest part of this project was the full flow. I built registration and login with JWT authentication, protected routes on the API side, and a dashboard that actually reflected reminder logic from the backend. The “Due Now” and “Upcoming” sections made the project feel like a real application instead of just a set of forms connected to a database.

Challenges and learning

A lot of the real work on this project was debugging. There were times where an endpoint worked perfectly in Postman but failed in the Flutter app, which meant tracing the problem through headers, tokens, emulator networking, state updates, and database records. That process taught me a lot more than just adding another feature would have.

Why it belongs in this portfolio

This is the clearest example of me building something end-to-end. It shows backend development, API design, data modeling, authentication, mobile UI work, and the ability to connect all of those parts into one functioning system.

Technologies

Flutter, Node.js, Express, Prisma, SQLite, JWT, Postman

Key strengths shown

Full-stack integration, debugging, API design, authentication, mobile development
