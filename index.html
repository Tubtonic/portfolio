About this portfolio

A quick overview

What this is

This portfolio was created for my final CPA portfolio presentation. It brings together projects from different parts of the program, including web development, testing, mobile development, database design, and cloud work.

What I focused on

Across these projects, I kept building the same core strengths: solving technical problems, working through real implementation issues, connecting systems properly, and thinking about how software supports actual user or business needs.

Selected work

Five projects from the program

01

Product Showcase Website

Next.js · React · Contentful · JavaScript

A web application built to display and manage product content using a headless CMS. This project involved dynamic content rendering, API integration, and building a cleaner structure for content-driven pages.

What I learned

I learned how a headless CMS changes the way content is managed, how to pull in external data properly, and how to work through API and permissions issues that come up in real projects.

Why it matters

This project shows my ability to build web applications that are not just static pages, but systems that work with external services and real content workflows.

02

Selenium Test Automation Project

Selenium IDE · Python · Test Automation

An automation project focused on reproducing and validating a user workflow on a web application. The goal was to turn manual testing steps into repeatable automated tests.

What I learned

I learned how important testing is outside of just writing code. It made me think more about user flows, edge cases, and how even small front-end changes can affect automation reliability.

Why it matters

This project shows that I understand software quality is part of development, not something separate from it.

03

KIT Relationship Manager

Flutter · Node.js · Express · Prisma · SQLite

KIT is a relationship management mobile app designed to help users keep track of people, log interactions, and stay on top of reminders. This is one of the projects that best represents where my skills are now.

What I learned

This project brought together frontend, backend, database, and authentication work in one system. It taught me how to build end-to-end functionality and how to troubleshoot issues between a mobile app, an API, and a local database-backed backend.

Why it matters

It shows that I can build a full system, not just one piece of one, and that I can work across different tools and layers.

04

Press Start Database Project

Database Design · ERD Modeling · SQL Concepts

A database design project for a small game store that needed support for sales, trade-ins, repairs, customers, employees, and inventory. The project focused heavily on structure and business logic.

What I learned

I learned how much thought has to go into database design before implementation even starts. It also helped strengthen my ability to reason through relationships, normalization, and the difference between transactional systems and analytics-focused systems.

Why it matters

This project shows my analytical side and my ability to design systems around real business operations instead of just technical theory.

05

AWS Cloud Project

AWS · EC2 · CloudFormation · Security Groups · Systems Manager

A cloud infrastructure project where I configured and validated AWS resources in a lab environment, including compute, access rules, and deployment-related setup.

What I learned

I learned how cloud setup works in practice and how easy it is for small configuration issues, especially networking and access rules, to stop a system from working the way you expect.

Why it matters

This project shows that I can move beyond application code and understand the infrastructure side of systems too.
