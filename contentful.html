Product Showcase Website

A full-stack style web project using Next.js and Contentful that helped push me toward more realistic web application structure.

Role
Individual project
Main stack
Next.js, React, Contentful, Tailwind CSS
What it taught me
Framework structure and CMS integration

Overview

This was a product showcase website built using Next.js, Tailwind CSS, Contentful CMS, and role-based authentication logic. The site displays products retrieved from Contentful, uses dynamic product pages, and includes a custom AuthContext and role-based access control.

I chose this project because it marked a point where my work started feeling more like actual application development rather than just putting features on a page.

Why this project is here:
I chose this project because it shows an important part of my development through the CPA program, whether that was building a complete system, improving how I structure code, understanding business requirements better, or learning from earlier mistakes.

What worked well

One of the biggest strengths of this project was the structure. I broke the work into clear sections: setting up Contentful, building the Next.js page structure, handling data fetching, creating authentication state, and protecting parts of the UI.

That made the project much easier to debug because I could isolate problems instead of trying to fix everything at once.

What was challenging

The biggest challenge was authentication and role-based access control. Managing state across refreshes and understanding how React context behaves took more effort than I expected.

I also underestimated how strict Next.js routing conventions are. Some of the errors came down to file placement, dynamic route structure, and environment configuration.

What the finished product does well

The application renders products dynamically from Contentful, uses scalable dynamic routing for product pages, keeps logic separated cleanly, and uses role-based protection for certain features.

I also think the project is more professional than some of my earlier work because of the cleaner folder structure and better handling of environment variables and sensitive keys.

What I learned

This project deepened my understanding of Next.js conventions, framework structure, CMS integration, and frontend state management.

It also reinforced secure development habits. Using .env.local properly and keeping API credentials out of the repository was one of those small things that makes a project feel more real.

How it reflects my growth

What I am proud of here is not just that it works, but that it feels organized. It shows progress from building something that simply functions toward building something that is structured, maintainable, and closer to how a real web project should be put together.
